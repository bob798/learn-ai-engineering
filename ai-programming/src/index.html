LEARNING GOAL
四个模块有内在的学习逻辑：先搭好工具环境，再理解编排框架，再看实战落地，最后把自己定位在六境路径上。
1
工具篇 — tmux + iTerm2 + Claude Code
从零搭建 AI 编程的终端工作台。学会 tmux 三层结构（Session / Window / Pane），配置 iTerm2 避开常见坑，理解 Claude Code 多实例管理的四种方式。环境不对，后续一切低效。
打开工具篇 →
2
OMC 深度拆解 — 多 Agent 编排框架
深入理解 oh-my-claudecode 的架构设计与 Agent 编排模式。从 Executor / Planner / Architect 角色分工，到 Skill 系统机制，到 Ralph / Autopilot 执行模式。是 L4 金丹境最重要的工程框架认知。
打开 OMC 拆解 →
3
实战案例 — PM 音频客服等
用真实项目检验理论。音曼客服三角色架构（接待 / 专家 / 验收）是多 Agent 协作的完整样本；企业 RAG 落地是 L3 筑基境知识的工程延伸。案例不只是看，要对照自己的项目找共鸣。
打开实战案例 →
4
六境修炼 — 定位自己，规划突破
回到总图，把自己的位置标出来。每个境界有明确的突破条件：能向外行清晰解释、能稳定复现预期输出、能独立上线真实应用、能交付自主完成复杂任务的 Agent。路径清晰，修炼不迷失。
