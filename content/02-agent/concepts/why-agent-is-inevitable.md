# Agent 是什么？为什么 AI 必然会走向 Agent

> 从达特茅斯会议到 GPT，再到 Agent，一篇文章看懂 AI 的演进逻辑。

---

## 先不解释 Agent，先看一个事实

1956 年，达特茅斯会议提出 AI。

2023 年，GPT 爆发，全民聊天。

2025 年，几乎所有 AI 公司都在做 Agent。

三个时间点，跨度近 70 年。中间换过好几拨技术——专家系统、机器学习、深度学习、Transformer、LLM——但方向一直没变。Agent 不是突然冒出来的新概念，它是这条线上的下一站。

问题是：凭什么是 Agent？它到底解决了什么问题，让整个行业同时转向？

---

## Agent 之前，AI 走过的路

先把时间线拉出来，看 AI 每一代在做什么、缺什么。

```
专家系统 (1970s-80s)    人写规则，机器照做
       ↓
机器学习 (1990s-2000s)  给数据，让机器自己找规律
       ↓
深度学习 (2012-)        更大的网络、更多的数据，效果起飞
       ↓
Transformer (2017-)     注意力机制，序列建模质变
       ↓
LLM (2020-)             大语言模型，能对话、能推理
       ↓
Agent (2024-)           能自己规划、调工具、完成任务
```

每一代比上一代多一层能力，但到 LLM 为止，有一个根本限制始终没突破。

---

## 回到第一性原理：AI 到底想做什么

达特茅斯会议的原始提案（1956）写得很直白：

> "尝试找到如何让机器使用语言、形成抽象与概念、解决目前只有人类才能解决的各种问题，并实现自我改进。"

注意这句话里没有"聊天"。AI 的目标，从来不是做一个对话框，而是模拟人的智能。

那什么叫智能？这才是关键问题。

---

## 连续追问：什么是智能

顺着"智能"这个词往下钻，问五层就见底。

**什么是智能？**
不是知道很多东西，是能用所知道的去达成目标。一个人背了整本百科全书但不会用，没人说他"智能"。

**达成目标需要什么？**
需要推理。看到问题，想出下一步该干嘛。

**光推理够不够？**
不够。想出来还得做出来。一个人想了一套完美计划但不动手，目标还是达不成。

**做完就行了？**
不行。做完得知道做得对不对。看到结果，判断离目标近了还是远了，然后调整。

**调整靠什么？**
靠记住之前干过什么、结果怎样。没有记忆，每次都从零开始，调整无从谈起。

五层问完，"智能"不再是一个模糊的词，它可以拆成六个具体能力：

| 能力 | 做什么 |
|---|---|
| 目标 | 知道自己要干嘛 |
| 感知 | 接收外部信息 |
| 推理 | 根据信息想出下一步 |
| 行动 | 把想的变成做的 |
| 反馈 | 看到结果，判断对不对 |
| 学习 | 记住经验，下次做得更好 |

这六条不是谁发明的框架，是从"什么叫智能"这个问题倒推出来的。少了任一条，都不算完整的智能。

---

## LLM 为什么不够

拿这六条去对 LLM，缺口一眼就看出来。

LLM 有什么：知识（训练数据里的）和推理（根据上下文生成下一步）。这两条它做得很好，好到让人觉得"AI 已经很强了"。

LLM 没什么：

| 能力 | LLM 的现状 |
|---|---|
| 感知 | 只能读文本（后来加了多模态，但仍是被动接收） |
| 行动 | 只能输出文本，不能操作任何外部系统 |
| 记忆 | 上下文窗口用完就忘，没有跨会话记忆 |
| 反馈 | 不知道自己的回答是对是错，没有验证环节 |

所以 LLM 本质上是一个"只能回答，不能做事"的系统。你问它"帮我订明天的机票"，它会告诉你怎么订，但它订不了。你问它"这段代码有 bug 吗"，它会分析，但它跑不了、测不了、改不了。

这不是模型不够大或训练数据不够多能解决的。这是架构层面的缺失——LLM 的输入是文本，输出也是文本，它跟外部世界之间没有通路。

---

## Agent 补齐了什么

Agent 不是一个全新的东西，它是在 LLM 外面包了一层，把缺的那几条能力补上。

| 人的能力 | Agent 怎么实现 |
|---|---|
| 看到环境 | 感知：读文件、看屏幕、接收 API 返回 |
| 想下一步 | 推理：LLM 本身的能力 |
| 列计划 | 规划：把大任务拆成小步骤 |
| 动手干 | 工具调用：调 API、跑代码、操作数据库 |
| 记住经验 | 记忆：短期（上下文）+ 长期（外部存储） |
| 看结果调整 | 反馈：观察工具返回，判断是否达成目标，修正下一步 |

串起来就是一个循环：

```
拿到目标
  ↓
想（推理 + 规划）
  ↓
做（调工具）
  ↓
看（观察结果）
  ↓
判断：目标达成了吗？
  ├── 没有 → 回到"想"
  └── 达成 → 结束
```

这个循环就是 Agent 的核心。它不复杂，但它解决了一个根本问题：让 AI 从"生成答案"变成"完成任务"。

如果你读过 ReAct 论文，会认出这就是 Thought → Action → Observation 循环。不是学术界凭空造了一个模式，而是"要完成任务"这个需求本身就要求这样的结构。

---

## 为什么 Agent 是必然演进

回到开头的问题：为什么 AI 一定会走向 Agent？

因为 AI 的目标从来就是模拟完整智能，而完整智能需要六种能力。LLM 只覆盖了其中两种（知识 + 推理），剩下四种（感知、行动、记忆、反馈）靠模型本身补不上，必须在模型外面加一层。这层东西加上去，就叫 Agent。

不是谁发明了 Agent 这个概念然后大家跟风。而是 AI 的能力积累到 LLM 阶段之后，下一步该补什么，倒推出来就是 Agent 这个形态。

换个角度看：

```
只能生成文本         → LLM
能调工具             → LLM + Tool Use
能根据结果调整       → LLM + Tool Use + 反馈循环
能记住跨会话经验     → LLM + Tool Use + 反馈循环 + Memory
能自己拆解复杂任务   → LLM + Tool Use + 反馈循环 + Memory + Planning
```

每加一层，离"完整智能"就近一步。加到最后，就是 Agent。这条路不存在岔道——你不可能跳过"行动"直接到"学习"，也不可能不要"反馈"就做好"规划"。能力之间有依赖关系，演进顺序几乎是定死的。

---

## Agent 之后还会怎么走

Agent 补齐了单个智能体的能力，但离"完整智能"还差两步。

**多 Agent 协作**：一个人能做的事有限，一群各有专长的人协作能做更多。现在已经有很多 Multi-Agent 框架在做这件事——让不同 Agent 分工、对话、互相审核。

**自主学习**：目前的 Agent 虽然有记忆，但还不能真正从失败中自我改进。它记住的是"上次用户说了什么"，而不是"上次我哪里做错了、下次该怎么改"。当 Agent 能做到后者，才算真正闭环。

```
Single Agent        一个人干活
     ↓
Multi-Agent         一群人协作
     ↓
Self-learning       从错误中自我改进
     ↓
AGI                 通用人工智能
```

Agent 不是终点，是 AI 从"语言模型"变成"智能系统"这条路上，走到现在的位置。

---

## 一句话收尾

LLM 的目标是生成答案，Agent 的目标是完成任务。AI 正在从"能说"演进为"能做"，Agent 是这条路上绕不过去的一步。
