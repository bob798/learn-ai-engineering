# 🎯 Prompt 收藏库

> 我的个人 prompt 仓库 —— 把"用过一次就忘"的高价值提问，沉淀成可复用的模板。

每个 prompt 都遵循统一结构，方便检索、复用、迭代：

- **触发场景** — 我在什么情况下会用它
- **一句话原话** — 我脑子里最初的那句话（口语版，方便联想）
- **优质 Prompt** — 打磨后的正式版本，直接复制即可使用
- **产出结构** — 期望 AI 输出长什么样
- **使用提示** — 怎么用效果最好

## 01 · Bug 解决过程回溯 · 提炼通用解法

**触发场景**
一个 bug / 棘手问题刚被我搞定（尤其是 **AI 一开始也解不出来、靠我自己拐过弯的那种**）。趁热把"我到底做对了什么"提炼出来，避免下次重新踩坑。

**一句话原话**
> 回溯下 bug 解决过程，遇到 AI 无法解决的问题，我做了哪些、怎么做的；这属于什么领域问题，通用解法是什么。

**优质 Prompt**

```text
请帮我复盘刚才这个问题的解决过程，目标是把这次经验提炼成"下次能直接复用"的知识。请严格按以下结构输出：

## 1. 问题快照
- 现象：一句话描述当时看到的错误/异常表现
- 真正的根因：剥离表象后，本质上是什么出了问题

## 2. AI 卡住的地方
- AI（包括你自己）最初给出的方向是什么、为什么不奏效
- 关键的误导点 / 信息盲区在哪里（是上下文缺失、还是问题被错误归类了？）

## 3. 人这一侧的破局动作
按时间顺序列出我（人类）做了哪些关键操作，并对每一步标注：
- 做了什么（具体动作）
- 为什么这么做（当时的判断依据 / 直觉）
- 这一步提供了什么 AI 之前拿不到的新信息

## 4. 领域归类
- 这个问题本质上属于哪个领域/类别（如：并发竞态、缓存一致性、编码/序列化、环境差异、依赖版本冲突、上下文窗口、提示词歧义……）
- 判断依据：它和这一类问题的共同特征是什么

## 5. 通用解法（最重要）
- 面对"这一类"问题的标准排查路径 / 解决套路（步骤化）
- 有哪些可复用的诊断信号、经验法则（rule of thumb）
- 下次遇到，我应该先问哪几个问题、先看哪几个地方

## 6. 一句话心法
用一句话总结这次经验，写成我能贴在笔记本扉页上的那种话。

要求：
- 诚实区分"我的功劳"和"运气成分"，别美化过程
- 第 5 部分要写成对"同类问题"都成立的通用解法，而不是只针对这一个 bug
- 如果信息不足以填某一节，直接说明缺什么，不要编
```

**产出结构**
六段式：问题快照 → AI 卡点 → 人的破局动作 → 领域归类 → **通用解法** → 一句话心法。重心在第 4、5 两段（从特例上升到通用）。

**使用提示**
- 最好在**同一段对话里**用 —— AI 还记得完整上下文，复盘最准。
- 如果是事后回顾，先把当时的错误日志 / 关键代码 / 对话记录贴回来再问。
- 第 5 段的"通用解法"值得单独存档；攒多了就是一份属于自己的《故障模式手册》。
